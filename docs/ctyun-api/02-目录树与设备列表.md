# 目录树与设备列表（精简版）

## 1. 获取监控目录树
接口：`/open/token/device/getReginWithGroupList`

### 关键点
- 只能获取当前层级的目录结构
- 需要根据 `hasChildren` 递归下钻
- 需要根据 `havDevice` 判断当前层级是否有设备
- 普通用户没有业务树权限，企业主账号才可正常使用

### 关键字段
- `id`
- `regionCode`
- `name`
- `level`
- `hasChildren`
- `havDevice`

## 2. 获取目录下设备列表
接口：`/open/token/device/getDeviceList`

### 关键点
- 只返回**当前目录层级**设备
- 需要把目录树递归和设备列表组合使用
- 如果要拿下级目录设备，必须继续对下级目录调用本接口

### 关键字段
- `deviceCode`
- `deviceName`

## 3. 项目规则
- 平台原始目录树只作后台同步来源
- 前台主结构不用平台原始目录树
- 前台按用户自定义目录组织点位
- 一个点位只能属于一个自定义目录

## 4. 项目实现建议
- 平台同步时先递归拉树，再分页拉设备
- 只把平台目录当来源，不直接暴露给前台
- 点位编码作为主键，设备名称只是平台同步字段
