# 99-接口清单总表

> 用途：给 Codex 在真正接平台接口时快速查完整 URL、请求方式、核心参数与解密规则。
> 
> 适用阶段：从“平台接入轮”开始使用。第 1 轮纯界面壳子可不依赖本文件。
> 
> 当前项目统一基线：
> - 网关域名：`https://vcp.21cn.com`
> - 请求方式：`POST`
> - `Content-Type`: `application/x-www-form-urlencoded`
> - 请求头固定：`apiVersion: 2.0`
> - 公共请求参数固定使用：`version = "1.1"`
> - 请求侧：私有参数拼装后放入 `params`，再用 **XXTea** 加密
> - 签名：`HMAC-SHA256(appId + clientType + params + timestamp + version, appSecret)`
> - 响应侧：按 `version = "1.1"` 统一使用 **RSA 私钥解密**（若接口返回的 `data` 已经是对象，则直接反序列化）
>
> 说明：本表只保留接第一阶段开发真正需要的接口；更细的字段说明仍以 `docs/ctyun-api/` 下分专题文档为准。

---

## 1. 鉴权与公共协议

### 1.1 获取 / 刷新访问令牌
- 接口名：`getAccessToken`
- 完整 URL：`https://vcp.21cn.com/open/oauth/getAccessToken`
- 用途：获取 `accessToken` / `refreshToken`，以及刷新令牌
- 关键参数：
  - `grantType`
    - `vcp_189`：用户无感知方式获取访问令牌
    - `refresh_token`：刷新访问令牌
  - `refreshToken`：刷新时必填
- 响应说明：通常直接返回 JSON 对象，可直接解析 `data.accessToken`、`data.refreshToken`、`expiresIn`、`refreshExpiresIn`
- 当前实现建议：本地缓存、过期刷新，不要高频重复申请

### 1.2 公共请求协议
- 通用域名：`https://vcp.21cn.com`
- 通用路径前缀：`https://vcp.21cn.com/open/token/`
- 公共参数：
  - `appId`
  - `version`
  - `clientType`
  - `timestamp`
  - `signature`
  - `params`
- 请求头：
  - `apiVersion: 2.0`
- 当前项目固定：
  - `version = "1.1"`
  - 请求参数 `params` 仍用 XXTea
  - 响应 `data` 优先按 RSA 私钥解密

## 2. 目录树与点位清单

### 2.1 查询监控目录树
- 接口名：`getReginWithGroupList`
- 完整 URL：`https://vcp.21cn.com/open/token/device/getReginWithGroupList`
- 用途：获取当前账号所分配的区域及其下级目录结构
- 关键参数：
  - `regionId`：为空时取首层
  - `accessToken`
  - `enterpriseUser`
  - `parentUser`（子账号场景）
- 返回重点字段：
  - `id`
  - `name`
  - `hasChildren`
  - `havDevice`
  - `regionCode`
- 当前实现建议：必须递归，不可只拉首层

### 2.2 查询监控目录设备列表
- 接口名：`getDeviceList`
- 完整 URL：`https://vcp.21cn.com/open/token/device/getDeviceList`
- 用途：获取某个目录层级下的设备列表
- 关键参数：
  - `accessToken`
  - `enterpriseUser`
  - `parentUser`
  - `regionId`
  - `pageNo`
  - `pageSize`
- 返回重点字段：
  - `data.list[].deviceCode`
  - `data.list[].deviceName`
- 当前实现建议：
  - 只能拿“当前目录层级”的设备
  - 要与目录树递归组合使用

### 2.3 查询目录设备总数 / 在线设备数（可选辅助）
- 接口名：`getCusDeviceCount`
- 完整 URL：`https://vcp.21cn.com/open/token/vcpTree/getCusDeviceCount`
- 用途：辅助做目录级统计
- 当前阶段：可后置，不必第一时间接入

## 3. 点位详情与状态

### 3.1 获取点位平台详情（主详情接口）
- 接口名：`getDeviceInfoByDeviceCode`
- 完整 URL：`https://vcp.21cn.com/open/token/device/getDeviceInfoByDeviceCode`
- 用途：作为点位详情主接口；同步平台名称、经纬度、位置、型号等
- 关键参数：
  - `accessToken`
  - `enterpriseUser`
  - `deviceCode`
- 返回重点字段：
  - `deviceCode`
  - `deviceName`
  - `deviceModel`
  - `longitude`
  - `latitude`
  - `location`
  - `isGbDevice`
  - `gbId`
  - `deviceType`
  - `gat1400Id`
  - `reportTime`
- 当前实现建议：
  - 点位主档同步时补拉本接口
  - 经纬度直接入库，为后续地图接入预留

### 3.2 批量查询设备在线状态（推荐主用）
- 接口名：`batchDeviceStatus`
- 完整 URL：`https://vcp.21cn.com/open/token/vpaas/device/batchDeviceStatus`
- 用途：巡检前快速获取设备当前状态
- 关键参数：
  - `accessToken`
  - `enterpriseUser`
  - `parentUser`
  - `deviceCodes`（单次最多 200）
- 状态说明：
  - `0` 离线
  - `1` 在线
  - `2` 休眠
  - `3` 休眠（保活）
- 当前实现建议：
  - 作为状态探测主接口
  - 只作辅助证据，不作为最终故障结论

### 3.3 批量查询设备状态（旧/补充）
- 接口名：`batchDeviceStatus`
- 完整 URL：`https://vcp.21cn.com/open/token/device/batchDeviceStatus`
- 用途：可查在线状态 / 云存状态 / 绑定状态
- 当前实现建议：第一阶段优先用 vpaas 版本，必要时再补本接口

### 3.4 单点位状态查询（可选）
- 接口名：`getDeviceStatus`
- 完整 URL：`https://vcp.21cn.com/open/token/vpaas/device/getDeviceStatus`
- 用途：单点位详情页即时刷新状态时可用
- 当前实现建议：不是第一阶段硬依赖，优先复用批量状态接口

## 4. 视频流与视频信息

### 4.1 获取 RTSP 直播地址
- 接口名：`getDeviceMediaUrlRtsp`
- 完整 URL：`https://vcp.21cn.com/open/token/cloud/getDeviceMediaUrlRtsp`
- 用途：获取点位 RTSP 直播地址，用于巡检复核与截图
- 关键参数：
  - `accessToken`
  - `enterpriseUser`
  - `parentUser`
  - `deviceCode`
  - `mediaType`
  - `mute`
  - `expire`
  - `netType`
- 响应说明：
  - `data` 为加密数据
  - 在当前项目基线下，按 `version = "1.1"` 统一使用 RSA 私钥解密
  - 解密后重点字段：
    - `url`
    - `expireTime`
- 当前实现建议：
  - 每次真正播放前重新取最新地址
  - 统一按 `rtp over tcp` 播放
  - 不缓存为长期固定地址

### 4.2 获取视频流信息
- 接口名：`getVideoInfo`
- 完整 URL：`https://vcp.21cn.com/open/token/vpaas/device/getVideoInfo`
- 用途：在成功拉流后记录视频编码、帧率、码率、分辨率、音频格式
- 关键参数：
  - `deviceCode`
  - `enterpriseUser`
  - `mediaType`
- 返回重点字段：
  - `codec`
  - `fps`
  - `bps`
  - `clarity`
  - `audioFormat`
- 当前实现建议：
  - 成功拉流后再调用
  - 作为诊断与展示字段，不作为最终故障唯一依据

## 6. 企业微信通知（当前阶段实现约定）

> 这部分不是天翼视联接口，但属于第一阶段必接外部能力。

### 6.1 当前项目固定通知样式
- 第 1 条：先 `@` 维护人手机号
- 第 2 条：发送单图海报
- 编码：简体中文 UTF-8

### 6.2 当前海报约定
- 三宫格截图
- 若只有 2 张有效图，第 3 格显示统一占位
- 若完全无法播放，三格都显示“无法正常播放”占位样式

### 6.3 当前字段来源优先级
- 点位名称：`DisplayName > PlatformDeviceName`
- `@` 目标：维护人手机号
- 经纬度：平台同步字段
- 巡检结论：最近巡检结果字段

## 8. 接口使用原则（给 Codex）

- 不要直接在页面里拼 HTTP 请求
- 必须统一走平台网关层
- 请求加密、签名、解密逻辑只实现一套
- 当前项目固定使用 `version = "1.1"`
- 启动时同步平台数据，巡检前再同步本次巡检范围内数据
- 平台原始目录仅作同步来源，前台界面使用用户自定义目录
- 最终故障判定以“能否稳定获得正常画面”为准
