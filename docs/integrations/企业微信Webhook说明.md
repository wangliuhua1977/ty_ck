# 企业微信 Webhook 说明（项目约定版）

## 1. 当前项目采用的通知方式
第一阶段只保留一套通知机制：
- 巡检触发通知
- 人工触发通知

两者复用同一套 webhook 输出逻辑。

## 2. 当前项目固定样式
发送顺序固定为：
1. 先 @ 维护人手机号
2. 再发送单图海报

## 3. 海报规则
- 海报主体为中文 UTF-8
- 截图区域固定三宫格
- 若只有 2 张有效截图，第 3 格使用统一占位
- 若完全无法正常播放，整张海报使用“无法正常播放”占位样式

## 4. 海报建议内容
- 点位名称（别名优先）
- 平台设备名称
- 经纬度
- 平台状态
- 归属单位
- 维护负责人
- 维护人手机号
- 宽带接入号
- 宽带接入号状态
- 巡检结论
- 2–3 张截图或无法播放占位

## 5. 维护人 @ 规则
- 不做 UserId 映射
- 直接使用维护人手机号作为 @ 目标
- 维护负责人姓名只用于展示
